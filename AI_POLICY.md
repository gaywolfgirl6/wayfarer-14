# AI-Assisted Development Policy

## Purpose

This project permits the use of AI-assisted tools (e.g., code generation, autocomplete, refactoring assistance) to support development. However, all contributions must meet the same quality, reliability, and maintainability standards as human-written code.

This policy outlines expectations for contributors using AI tools.

---

## Guiding Principles

AI is a tool, not a source of truth.

All AI-generated or AI-assisted output should be treated as **potentially incorrect, incomplete, or insecure** until proven otherwise. Contributors are fully responsible for the code they submit, regardless of how it was produced.

## Acceptable Use

AI-assisted development is allowed under the following conditions:

- AI output must be **reviewed, understood, and validated** by the contributor before submission.
- Code must be **tested rigorously**, including:
  - Functional correctness
  - Edge cases
  - Performance considerations (where applicable)
  - Security implications
- The contributor must be able to **explain the code they are submitting**.

Blindly copying and submitting AI-generated code is not acceptable.

## Accountability

Contributors are fully accountable for:
- Bugs
- Security vulnerabilities
- Performance regressions
- Maintainability concerns

"Attributing issues to AI" is not an acceptable justification for poor quality.

## Enforcement

To protect the integrity of the codebase:

- Repeated submission of low-quality, unreviewed, or problematic AI-assisted code may result in:
  - Increased review requirements
  - Temporary contribution restrictions
  - Removal of commit access ("git ban") in severe or repeated cases

These actions are not punitive, but are intended to maintain project quality and stability.
